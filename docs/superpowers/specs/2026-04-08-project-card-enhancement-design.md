## 概述

增强首页项目卡片，在现有（项目名称、最后编辑时间）基础上，新增：阶段进度指示器、成员头像、项目类型、自定义标签，并配套新建弹窗和卡片菜单的编辑入口。

---

## 数据模型

在 `src/lib/storage.ts` 的 `Project` interface 新增以下字段：

```ts
export type ProjectStage = 'script' | 'storyboard' | 'generation' | 'review';
export type ProjectType = '短片' | '广告' | 'MV' | '教程' | '其他';

// 新增字段
stageOverride?: ProjectStage;  // 用户手动覆盖阶段；undefined 表示使用自动推断
members: string[];             // 成员姓名数组，如 ['张三', '李四']
projectType?: ProjectType;     // 固定分类
tags: string[];                // 自定义标签
```

`createProject()` 初始化时，`members` 和 `tags` 默认为空数组。

### 阶段自动推断

提供纯函数 `inferStage(project: Project): ProjectStage`，优先返回 `stageOverride`，否则按以下顺序推断：

| 条件 | 推断阶段 |
|---|---|
| `videoOrder.length > 0` | `'review'` |
| 存在 `type === 'imageNode'` 且 `data.content` 非空的节点 | `'generation'` |
| `storyboardRows.length > 0` | `'storyboard'` |
| 其他 | `'script'` |

## 卡片 UI（`ProjectCard` 组件）

### 信息布局（从上到下）

1. **缩略图区域（上方 ~60%）**
   - 左上角：项目类型 badge（`短片` / `广告` 等），半透明背景，低饱和度边框
   - 无类型时不显示 badge

2. **信息区域（下方）**
   - 项目名称（粗体，可双击编辑，已有逻辑保留）
   - 最后编辑时间（`timeAgo`，已有逻辑保留）
   - 自定义标签列表（小圆角 pill，低饱和度，无标签时占位高度 `min-height` 保持布局稳定）
   - 阶段步骤连线（4 个圆点 + 3 条连线）
   - 底部行：成员头像（叠排）+ `···` 菜单按钮

### 阶段步骤连线视觉规则

- 已完成阶段：`rgba(200,190,220,0.7)` 填充圆，内显示 `✓`，连线用 `rgba(200,190,220,0.35)`
- 未完成阶段：`rgba(255,255,255,0.07)` 填充 + `rgba(255,255,255,0.15)` 边框，连线用 `rgba(255,255,255,0.1)`
- "已完成"定义：`inferStage` 返回的阶段 ≥ 该步骤（script=0, storyboard=1, generation=2, review=3）

### 成员头像

- 叠排显示，每个头像取姓名首字（中文取第一个字，英文取首字母大写）
- 颜色：从预设低饱和度色板按成员索引取色（如 `rgba(160,145,130,0.45)` / `rgba(145,155,165,0.45)` / `rgba(130,150,135,0.45)` 等）
- 超过 3 人显示 `+N`
- 无成员时显示一个透明占位圆

## 新建项目弹窗

现有的"新建项目"逻辑在 `HomePage` 中直接调用 `onNewProject()`，扩展为弹出对话框，字段顺序：

1. **项目名称**（文本输入，默认"未命名项目"）
2. **项目类型**（pill 单选：短片 / 广告 / MV / 教程 / 其他）
3. **标签**（输入框，回车添加，显示已添加标签 + × 删除）
4. **成员**（输入框，回车添加姓名，显示头像列表 + × 删除）
5. **取消 / 创建** 按钮

取消：关闭弹窗，不创建  
创建：`onNewProject` 的调用方需将 Props 签名从 `onNewProject: (initialScript?: string) => void` 扩展为接受初始项目元数据，或在弹窗内部先组装 `Partial<Project>` 再调用。推荐方案：`HomePage` 内部处理弹窗状态，弹窗确认后直接调用 `createProject()` 并传入额外字段，再回调 `onNewProject`。

## 卡片菜单（`···` hover 菜单）

在现有"重命名"和"删除项目"基础上新增三项：

| 菜单项 | 行为 |
|---|---|
| 编辑标签/类型 | 打开标签+类型编辑小弹窗 |
| 管理成员 | 打开成员管理小弹窗 |
| 调整阶段 | 展开子菜单（内联） |

### 编辑标签/类型弹窗

- 顶部：项目类型 pill 单选（同新建弹窗，当前选中高亮）
- 下方：标签输入框（回车添加，显示已有标签 + × 删除）
- "完成"按钮保存，变更立即写入 `project.projectType` 和 `project.tags`

### 管理成员弹窗

- 列出现有成员（头像 + 姓名 + × 删除按钮）
- 输入框：输入姓名后回车添加
- "完成"按钮保存

### 调整阶段子菜单

- 内联展开，显示 4 个阶段选项
- 当前阶段高亮，已完成阶段显示 `✓`
- 选中后写入 `stageOverride`
- 底部"恢复自动推断"选项：清除 `stageOverride`（置为 `undefined`）
