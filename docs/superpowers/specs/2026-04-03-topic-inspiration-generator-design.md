# 选题灵感发散器 · 设计文档

**日期：** 2026-04-03
**功能名称：** 选题灵感发散器（Topic Inspiration Generator）
**状态：** 待实现

---

## 一、功能概述

在现有 AI 内容创作工作台中新增"选题"Tab，位于底部导航栏最左侧（拆本之前）。

用户输入题材关键词，系统通过豆包联网插件搜索 B站、小红书、抖音等平台的相关视频内容，分析互动数据、内容摘要和评论区，输出：
1. **爆款配方**：该题材高互动内容的共同规律（痛点、钩子、情绪、结构）
2. **选题建议**：6-8 个具体可执行的视频标题/角度，附带共鸣原因和情绪标签

用户从建议中选定或自由编辑想法，可存档至项目，也可一键导入拆本流程。

---

## 二、产品定位与工作流位置

```
选题（新） → 拆本 → 画板 → 分镜 → 视频 → 字幕 → AI对话
```

选题是创作的起点，解决"不知道做什么"的问题，为后续拆本提供有数据支撑的输入。

---

## 三、页面布局

页面分为上中下三个区域：

### 3.1 顶部：输入区

- 文本输入框：`输入题材关键词，如"职场穿搭"、"独居生活"…`
- 平台多选 Toggle：`[B站] [小红书] [抖音]`（默认全选，可单独取消）
- 主操作按钮：`开始分析`（loading 状态显示进度文案）

### 3.2 中部：分析结果区（左右分栏）

**左栏：数据概览 + 视频列表**

数据概览卡片组（仿竞品但精简）：
- 平均播放量（含中位数、最大值）
- 平均点赞量
- 平均收藏量
- 评论关键词（TOP 3 情绪词）

视频列表表格：
- 列：缩略图 / 标题（可点击原链接） / 平台标识 / 播放量 / 点赞 / 收藏 / `深度解析` 按钮
- 点击`深度解析`：该行下方展开折叠面板，显示：
  - AI 归纳的"这条视频好在哪里"（3-5 条）
  - 代表性评论摘录（3 条）
  - 情绪标签（感动/共鸣/涨知识/争议等）

**右栏：AI 洞察报告**

- **爆款配方**：结构化要点列表（流式打字输出）
  - 内容切入角度
  - 情绪钩子类型
  - 标题结构规律
  - 高频评论诉求

- **选题建议**：6-8 张卡片，每张包含：
  - 建议标题
  - 共鸣原因（1-2 句）
  - 情绪标签 Badge
  - `采用` 按钮 → 填入底部编辑器

### 3.3 底部：想法编辑器

- 可编辑文本域（支持手动输入或点击"采用"自动填入）
- 操作按钮：
  - `存档` → 通过 WebSocket 保存到项目 `topicDraft` 字段
  - `→ 导入拆本` → 切换至拆本 Tab，并将内容注入脚本输入框

## 四、数据流与后端设计

### 4.1 API 接口

**POST `/api/topic-research`**

请求体：
```json
{
  "keyword": "职场穿搭",
  "platforms": ["bilibili", "xiaohongshu", "douyin"]
}
```

返回（Server-Sent Events 流式）：
```json
// Phase 1：视频数据（非流式，一次性返回）
{
  "type": "videos",
  "data": {
    "summary": { "avgViews": 125800, "avgLikes": 7002, "avgFavorites": 4335 },
    "videos": [
      {
        "title": "视频标题",
        "platform": "bilibili",
        "url": "https://...",
        "thumbnail": "https://...",
        "views": 840300,
        "likes": 42500,
        "favorites": 27200,
        "brief": "内容简介",
        "topComments": ["评论1", "评论2", "评论3"],
        "analysis": "AI 归纳的爆款原因"
      }
    ]
  }
}

// Phase 2：AI 洞察（流式 token by token）
{ "type": "insight_chunk", "data": "..." }

// Phase 3：选题建议（完整 JSON，最后返回）
{
  "type": "suggestions",
  "data": [
    {
      "title": "建议标题",
      "reason": "为何引发共鸣",
      "emotionTag": "共鸣"
    }
  ]
}
```

### 4.2 后端实现

服务端新增路由 `POST /api/topic-research`（位于 `server/index.ts`）：

1. 构造第一轮 Prompt，调用豆包 ARK API（启用 `web_search` 插件）：
   - 搜索关键词：`${keyword} site:bilibili.com OR 小红书 高播放 评论`
   - 解析返回结果，提取视频摘要、互动数据、代表评论
   - 对每条视频做单独 AI 分析（并发，不超过 5 条）

2. 第二轮调用：综合所有视频信息，生成：
   - 爆款配方（流式 SSE 推送）
   - 选题建议列表（结构化 JSON）

3. 错误处理：
   - 联网搜索无结果 → 返回 `{ error: "no_results", message: "未找到相关内容，请换词重试" }`
   - API 超时（>30s）→ 返回已获取的部分结果，附带 `partial: true` 标识

## 六、交互细节

- **加载体验：** 分析过程中显示骨架屏占位，AI 洞察区流式打字输出，避免白屏等待
- **深度解析：** 点击后原地展开（不弹窗），再次点击收起
- **采用选题：** 点击卡片上的`采用`按钮，标题填入底部编辑器，用户可继续手动修改
- **导入拆本：** 切换 Tab 的同时向 `BreakdownView` 传入文本内容，填入其脚本输入框
- **存档：** 通过现有 WebSocket `wsSaveProject` 机制保存 `topicDraft` 字段，不新建存储层
- **平台图标：** B站蓝色、小红书红色、抖音黑色，视频列表行显示平台 Badge

---

## 七、不做的事（YAGNI）

- 不做跨项目的选题历史管理（初期只存在当前项目内）
- 不做爬虫或第三方数据平台 API 集成
- 不做视频真实互动数据的精确抓取（豆包联网返回的是摘要性数据）
- 不做多轮研究 Session 管理（方案二，留到后续版本）
- 不做选题的协作评论/打分功能

---

## 八、验收标准

1. 输入关键词 + 选平台后点击"开始分析"，30 秒内出现视频列表和 AI 洞察
2. 每条视频点击"深度解析"可展开 AI 分析内容
3. 点击任意选题建议的"采用"，内容填入底部编辑器
4. 点击"存档"后刷新页面，编辑器内容保留
5. 点击"导入拆本"后切换到拆本 Tab，脚本输入框已填入内容
6. 联网搜索无结果时显示友好提示，不报错崩溃
